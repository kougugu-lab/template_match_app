# CHANGELOG

## [2026-04-06] v1.3.0 - 画像処理機能の統合・標準化とUI洗練

### 追加 (Added)
- **画像前処理の一括設定**: 設定ダイアログの画像処理タブに「明るさ・コントラスト・彩度・ガンマ・ぼかし・シャープ・CLAHE」の7項目を追加。これらはリアルタイム検査にも適用されます。
- **エンジンへの前処理実装**: `engine.py` に OpenCV ベースの共通前処理 `apply_preprocessing` を実装。

### 修正 (Fixed)
- **設定画面内の TypeError**: `slider` 関数の引数名の不一致によるクラッシュを修正。
- **エディタ画面のクラッシュ**: 射影変換のリセットやマウス操作時に `image_x` 等の変数が未定義でエラーになる問題を解消。
- **InspectionEngine 呼び出しエラー**: インスタンス化の際の引数不足/過多を修正し、前処理プレビューが正常に機能するよう改善。

### 変更 (Changed)
- **前処理ロジックの標準化**: エディタ画面（PILベース）と検査エンジン（OpenCVベース）の画像補正処理を OpenCV に統一。プレビューと検査結果の一貫性が向上。
- **UIラベルのクリーンアップ**: 画像処理タブおよびエディタ画面から不要なカッコ書き（`(ROI)`, `(輪郭フィルタ)` 等）を削除し、名称を整理（単位は維持）。
- **ボタン配色の標準化**: `inspection_app` に合わせ、「保存・登録・開始・実行＝緑 (COLOR_OK)」、「削除・停止・リセット＝赤 (COLOR_NG)」に統一。
- **システム設定の整理**: 重複していた「輝度正規化 (CLAHE)」フラグを画像処理タブのスライダー形式に統合。

## [2026-04-05] v1.2.1 - UI/UX全面刷新とシステム利便性の向上

### 追加 (Added)
- **エディタ画面のツールチップ**: すべてのボタンやスライダーにマウスオーバー時の説明文（Tooltip）を追加。
- **データ拡張の出力先調整**: デフォルトの出力フォルダを `./master_image/augmented/` に設定し、マスター画像との混在を防止。
- **カメラ全項目自動設定ボタン**: 露出・フォーカス・ホワイトバランス・色相を一括で順番に自動走査・最適化する機能を実装し、カメラ設定タブの最上部に追加。
- **ログの自動クリーンアップ**: 画像ファイルと同様に、「30日以上経過した古い .log ファイル」を容量監視モジュール（`_monitor_storage`）が自動的に削除する機能を追加し、完全なメンテナンスフリーを実現。
- **多重実行防止ロック**: データ拡張実行中に「データ拡張を実行」ボタンを無効化（DISABLE）し、悪意や事故による連打でマシンがクラッシュするのを防止。
- カメラの自動再接続ロジックをメインのプレビューループに追加。
- バックアップ機能: 設定保存時に `config.json` を上書きする前に `config.json.bak` を自動で退避生成。
- `VALID_BCM_PINS` リストに基づく、強力なGPIOピン番号の存在チェックバリデーションを導入。
- **編集モードに「カメラから撮影」ボタンを追加**: 編集モード（EditorView）の「1. 読込・射影変換」セクションに「カメラから撮影」ボタンを新設。メインの検査プレビューが取得している最新フレームを瞬時にエディタに取り込み、即座に作業に入れるよう改善。
- **画像処理の全自動調整ボタン**: 「画像処理」タブに「画像処理設定を全自動調整」ボタンを新設。現在の映像を解析し、二値化モードから閾値・フィルタ範囲までの一括最適化をワンクリックで実行可能にしました。
- **エディタのスクロール機能**: 「編集ツール」サイドバーが低解像度ディスプレイで見切れる問題に対応。キャンバスによるスクロール領域を実装。

### 変更 (Changed)
- **画像処理設定のワークフロー最適化**: 「詳細設定 > 画像処理」タブの項目を、実際の処理順序（ROI → 二値化 → フィルタ → 補正 → 判定）に並べ替え。
- **エディタ操作の最適化**: 編集ツールの並び順をマスター作成フローに合わせて再定義。
- **マスターフォルダ仕様の変更**: 車種（Car Model）の概念を削除し、一貫して `./master_image/` を使用するシンプルな構造に移行。
- **UIのさらなるシンプル化**: 各タブのラベルから `(BCM番号)` や `(推奨)` などの不要な括弧書きを全面的に削除。
- **AI自動調整ボタンの再配置**: 画像処理タブの二値化セクションにおいて、最もアクセスしやすい最上部へボタンを移動。
- **仕様IDのソート表示**: 設定画面の仕様マッピングリストをIDの数値昇順（1, 2, 3...）で表示するように変更。
- デバッグ画像保存によるストレージの急激な使用増加を防止するため、標準ではデバッグ画像を保存しない仕様に変更し、システムタブに設定を新設。
- 無駄なループCPU使用を防止するため、調整タブがアクティブでないときの不要なUI更新ルーチンを徹底排除。
- **UI表示の日本語化徹底**: 「Ada Block」「Ada C」「Pi 40Pin Map」などを「適応ブロックサイズ」「適応定数C」「Pi 40ピン配置図」といった日本語名称に統一。
- **マウスホイールの挙動適正化**: サイドバー上ではスクロール、画像エリア上ではズーム、とホイールの役割を自動的に切り替えるように改善。
- **二値化モードの完全一元化**: 従来、画像処理タブとシステムタブで矛盾して設定できた二重構造を廃止し、3択ラジオボタンに統一。
- **`config.json` の自動クリーンアップ**: 起動時に旧バージョンで使用していた不要なキー（`mask_lh_up`, `THRESHOLD_FLAG` 等）を自動削除するメソッドを実装。

### 修正 (Fixed)
- **設定画面起動時のクラッシュ**: 削除済みの車種設定ウィジェットへの参照が残っていた問題を修正。
- **設定タブ・エディタの表示を改善**: 低解像度モニタでも項目が右側で見切れないよう、左側パネルの最小幅を `550px` に拡大しスクロール追従を強化。
- **変数定義漏れの修正**: 検査エンジン内のフォールバック処理において、未定義の変数を参照していた問題を修正。
- **カメラ初期化の致命的エラーを修正**: Windows環境で `cv2.CAP_DSHOW` 使用時にクラッシュが発生する問題に対し、ガードを強化し自動フォールバックを実装。
- **データ拡張の保存先同期**: データ生成（Augmentation）のデフォルト出力先を、現在選択されているマスター画像フォルダに自動的に合わせるよう修正。
- **UIラベルの洗練**: 各設定画面および仮想GPIOパネルから `(BCM)` や `(トリガー)` といった不要な括弧表記を削除。
- **GPIO設定画面の見切れ修正**: ラベル幅とパネル比率をさらに最適化し、右端の「削除」ボタンが隠れないようマージンを確保。
- **設定後のステータス表示バグ**: 詳細設定画面を閉じた後、メイン画面のステータスバーが「設定変更中」で止まる問題を修正。
- **出力ピン重複チェックの仕様変更**: 仕様マッピングの重複ピンチェックを緩和し、部分的なピン共有を許可。
- **設定キャンセル時のグローバルフリーズの修正**: 詳細設定画面を OS の「Xボタン」等で閉じた際のカメラ永続フリーズ等を修正。
- **Windows日本語パスクラッシュの修正**: OpenCV で日本語が含まれるパスを読み書きする際のクラッシュを回避。
- **データ拡張のフリーズ防止**: バックグラウンド処理中の例外キャッチと復帰ロジックの改善。
- 詳細設定タブを開いた際のバックグラウンドGPIOリソース競合を修正。
- 調整タブの重複生成されていた Checkbutton ウィジェットを削除。
- ROIが 640x480 でハードコード保存されていた問題を修正し、相対比率 (0.0~1.0) での保存に対応。
- **カメラ接続エラーの修正**: Windows環境で DShow 失敗時に自動で `CAP_ANY` へフォールバック。
- カメラ自動調整ボタン（オートチューニング）の重複実行バッファリング問題を修正。

### 削除 (Removed)
- **車種設定機能**: 未使用のまま残っていた車種選択・管理ロジックを完全に削除。

## [2026-04-04] v1.1.0 - UI/UX全面刷新および標準化

### 追加 (Added)
- **メインUIの改善**: 設定画面を開いている間、メイン画面のカメラプレビューをアイドリングさせ、処理負荷を軽減する機能を追加。
- **仕様マッピングの動的UI追加**: 仕様IDを動的に追加・削除できる最新の `inspection_app` 準拠のリストUIへ刷新。
- **カメラ自動調整機能の拡充**: 「画像処理タブ」および「カメラタブ」に自動最適化探索ボタンを追加。
- **マウスドラッグによる直感的なマスク（ROI）指定**: プレビューキャンバス上でマウスをドラッグして検査エリア（ROI）を指定できるよう簡略化。
- **仮想GPIOパネルの最新化**: Windows実行時に表示される仮想GPIOパネルのデザインを近代化。
- **GPIOテスト機能**: 各出力ピンの横に「テスト」ボタンを追加。
- **40Pin Map 入力の連動強化**: ピン配図をクリックした際、入力欄に自動的にピン番号がセットされるよう改善。
- **UI/UX 標準化の徹底**: メイン画面・設定画面の全てのコントロールにツールチップを追加し、共通要件に準拠。
- **GPIO保存バリデーション**: 重複ピンや範囲外の入力を保存時に検出し、警告する仕組みを導入。
- **マウスホイールスクロール**: 全ての設定タブにホイール操作による上下スクロールを実装。
- **包括的なツールチップ**: 設定項目の各入力フィールドに詳細な機能説明を表示するツールチップを配備。
- **統合アプリ TM_App 初版リリース**: (v1.0.0) 4つのスクリプトを統合し、ConfigManager、検査エンジン、マスター画像エディタ、5タブ設定ダイアログを実装。

### 変更 (Changed)
- **UIレイアウトの標準化**: `inspection_app` に準拠したダークテーマとカードレイアウトへ刷新。
- **GPIOラベルの適正化**: 「トリガー入力」「OK出力」「NG出力」へ名称変更。
- **解像度設定タブの集約**: 設定を「画素数・保存」タブに整理。
- **デフォルト設定**: プレビューFPSを 10 から 5 に変更。
- ヘッダーに判定状態別のカラー適用を実装。
- 設定ダイアログの変更検知ハイライト（保存ボタンの色変化）機能を実装。
- リアルタイム調整を「調整タブ」に集約。
- データ拡張処理のスレッド化とプログレスバー表示を実装。
- Windows用仮想GPIOパネルを統合。

### 修正 (Fixed)
- 設定画面で一部のUI属性が参照できず、保存が失敗していた不具合を修正。
- 複数の設定変更が重なった際の UI 更新ロジックの冗長性を解消。
