# CHANGELOG

## [2026-04-14] v1.4.0 - カメラ安定化、UIの洗練、および設定項目の最適化 (Current)

### 追加 (Added)
- **カメラ初期化の堅牢化**: OpenCVのバックエンド多重試行ロジック（MSMF → DSHOW → AUTO）を導入。Windows環境特有の C++ 例外による起動失敗を回避。
- **包括的なツールチップ実装**: 全入力フィールド、ボタン、スライダーに機能説明を表示するツールチップを追加。
- **プロパティ設定の保護**: カメラの各パラメータ（露出、フォーカス等）設定時に個別の例外ハンドリングを追加し、特定機能非対応のハードウェアでも安定稼働を実現。

### 変更・最適化 (Changed & Optimized)
- **UI表記のクリーンアップ**:
    - 「画素数」「システム」タブ等の各カードタイトルからナンバリングを削除し、視覚的なノイズを低減。
    - 画像処理ラベル等から不要な括弧書きや単位 `(mm)`, `(%)`, `(CLAHE)` を削除。
- **タブ構成の整理**:
    - 「画素数」タブを新設（旧「画素数・保存」から変更）し、解像度設定に特化。
    - 「画像結果保存先」の設定をシステムタブへ集約し、設定箇所を一元化。
- **規定値の変更**: ディスク容量管理のデフォルト上限値を 10GB から **30GB** へ引き上げ。
- **カメラ設定UIの視認性改善**: プロパティ入力幅を最適化し、`自動調整` ボタンを同一行に再配置。全自動ボタンの色味を学習ボタンと統一。
- **Windows実行時のGPIO接続挙動を改善**: `pigpio` 接続エラーメッセージの大量出力を回避し、モック動作へ安全にフォールバック。

### 修正 (Fixed)
- **カメラプレビューのクラッシュ解決**: 設定ダイアログで発生していた OpenCV の内部例外（C++ Exception）をバックエンドの適切な選択によって解消。
- **エディタ画面の NameError 修正**: 輪郭採用時に `InspectionEngine` が参照できずクラッシュする不具合を修正。
- **安定性向上**: `SettingsDialog` 内の変数初期化タイミングを適正化し、タブ切り替え時の安定性を向上。
- **カメラ再接続時の例外修正**: カメラ未接続/再接続失敗時に `NoneType.isOpened()` でプレビュースレッドが落ちる不具合を修正。
- **設定表示の視認性修正**: 「画素数」「保存先」項目で設定値が見えづらくなるケースに対し、Combobox表示スタイルと値反映を補強。
- **保存設定の初期値修正**: 結果保存先のデフォルトを `./results` に統一。
- **初期期待仕様表示の修正**: 起動直後に `期待: 不一致` と表示される問題を抑制し、未確定時は `期待: --` を表示。
- **設定保存時の体感遅延を軽減**: 保存後のハードウェア再初期化を非同期化し、UIのブロッキングを抑制。
- **設定正規化の強化**: 起動時に `camera/gpio/pattern/storage` の壊れ値や不整合を補正し、設定起因の異常を抑制。
- **容量監視ロジックの一本化**: `app.py` 内の重複実装を統合し、画像削除対象（`images`/`debug`）とログ保持ルール（30日）を明確化。
- **検査ループの安定化**: プレビュー取得を `grab/retrieve` に変更し、検査処理後に滞留トリガーを排水して遅延蓄積を抑制。
- **システム設定の拡張**: `preview_fps` / `ok_output_time` / `ng_output_time` / `result_display_time` を設定画面から変更可能にし、運転時チューニング性を向上。
- **出力保持時間の設定対応**: OK/NG 出力時間を設定値で制御するよう変更（NGは0秒で保持、正数で自動OFF）。
- **保存解像度キー互換**: `res_skip` / `res_record` を設定正規化と保存処理に追加し、既存の `res_ok` / `res_ng` と併用可能に統一。
- **推奨初期値を調整**: `camera.fps=10`, `preview_fps=12.0`, `ok_output_time=0.2`, `result_display_time=1.5` に見直し、初期運用時の応答性と視認性のバランスを改善。
- **運転プリセット追加**: `runtime.operation_preset` (`standard` / `accurate` / `fast`) を導入し、システムタブから推奨値を即時適用可能に。
- **環境プロファイル自動化**: `runtime.environment_profile` (`auto` / `windows_dev` / `raspi_prod`) と `auto_apply_environment` を追加し、環境に応じたFPS上限を自動適用。
- **互換キー統廃合の段階運用**: 保存解像度は `res_skip/res_record` を正として運用し、`compat_cleanup_enabled` 有効時は期限到達後に `res_ok/res_ng` を削除可能に。
- **プリセット適用仕様を修正**: 起動時の一律上書きを廃止し、`operation_preset` 変更時のみ推奨値を反映するよう改善（手動チューニング値を維持）。
- **互換キー削除の保存挙動を修正**: `compat_cleanup_enabled` かつ期限到達時は、保存処理でも `res_ok/res_ng` を再生成しないよう統一。
- **プリセット反映範囲を拡張**: `ng_output_time` と `camera.fps` も「推奨値を適用」で反映。
- **GPIO解放処理の不具合修正**: `pattern_inputs` を辞書値で確実に close するよう修正し、設定再読込時の残留リスクを低減。
- **設定保存の堅牢化**: システムタブの数値入力を安全変換に変更し、空欄/不正入力で保存が落ちないよう改善。
- **UI/UX改善（inspection_app準拠）**: システムタブを「運転プリセット・環境 / AI判定・表示 / 出力制御」に再編し、`fps/sec` の単位表示を明示。
- **保存解像度候補の最適化**: 撮影解像度より大きい保存候補を自動除外し、アップスケール設定の誤選択を抑制。
- **再撮影設定を拡張**: `max_retries` / `burst_interval` をシステムタブで設定可能にし、未検出時の連続試行を制御可能に。
- **検査ループ改善**: 未検出時に `max_retries` 回まで最新フレームで再試行する処理を追加し、取りこぼし耐性を向上。
- **NG出力時間の運用改善**: `ng_output_time` は空欄入力を許可し、空欄時は「ブザー停止」まで保持する仕様へ変更。
- **判定しきい値の導線追加**: システムタブから `decision_threshold` を直接調整可能にし、現場での感度調整を簡易化。
- **保存前バリデーション可視化**: システム系入力の範囲外/不正値で対象欄を赤枠表示し、エラー箇所を即時把握できるよう改善。
- **運転プリセット関連UIを削除**: 「運転プリセット・環境」設定項目を廃止し、手動設定ベースへ一本化。
- **運用初期値を変更**: `preview_fps=2`, `result_display_time=5`, `max_retries=10`, `burst_interval=0` を既定値へ反映。
- **`.bak` 運用を廃止**: 設定保存時の `config.json.bak` 自動作成を停止し、既存の `.bak` ファイルを削除。

## [2026-04-13] v1.3.3 - UI/UXの刷新、内部データ構成の現代化、操作感の最適化

### 追加 (Added)
- **リストベースのパターン管理**: `inspection_app` に準拠した、動的なパターン追加・削除・名称編集機能。
- **パターンごとのGPIO入力条件設定**: 各パターンに対し、入力ピン（複数可）の ON/OFF 組み合わせをトグル操作で設定できる直感的なUIを導入。
- **GPIO 40ピン配置図からの入力**: 設定画面のピン番号入力欄をクリックしてから配置図のピンをクリックすることで、BCM番号を自動反映する機能を実装。
- **データ構造の自動移行**: 旧バージョンの `specification_mapping` 形式から、新形式の `gpio` および `patterns` 構造へ起動時に自動変換するロジックを搭載。
- **仮想GPIOパネルの拡張**: パターン切替入力をシミュレートするためのチェックボックス群を仮想GPIOパネル（Windows用）に追加。
- **パターン照合タブの新設**: 外部からのGPIOパターン入力（HIGH/LOWの組み合わせ）と「期待する仕様」を紐付けるUIを新規に追加。
- **入力ベースの判定判定**: 従来の「結果ごとの出力ピン別振分け」仕様を廃止し、外部信号のパターンから「期待仕様」を読み取り、画像判定結果と内部で照合して OK や NG 信号を出力するアーキテクチャへと刷新。
- **判定結果の並立表示**: メインUIパネルに「期待仕様」と「最終判定（画像判定）」の両方を表示し、検証作業を一目で把握できるように改善。

### 変更・削除 (Changed & Removed)
- **GPIO設定タブの標準化**: `inspection_app` に合わせ「入力設定」「ピンマップ」「出力設定」のレイアウトを最適化。
- **内部通信プロトコルの刷新**: ハードウェア初期化およびパターン照合ロジックを辞書ベースへ移行。
- **表記の統一**: 「Pin」を「ピン」、「新規パターン」を「パターン」に統一。絵文字を削除しプロフェッショナルな外観へ。
- **スライダーからスピンボックスへの復旧**: 数値入力の正確性を高めるため、主要な調整項目にスピンボックスを再導入。
- **エディタ動作の超高速化**: 編集モードで「カメラから撮影」ボタンを押した際や、スライダーを動かした際に発生していた深刻なフリーズ・もっさり感を解消。画像処理エンジンの初期化再ロードを回避し、プレビュー描画アルゴリズムを最適化。
- **レガシー設定の撤廃**: 従来の個別仕様出力ピンの割り当てを `config.json` からクリーンアップ・削除。

### 修正 (Fixed)
- **設定画面の下部ボタン表示バグ修正**: 保存ボタンが画面外に押し出されて消えてしまうレイアウトの不具合を修正。
- **自動調整ボタンの視認性改善**: カメラタブの「全項目を自動調整」ボタンがスクロール領域に隠れる不具合を修正。
- **タブ切り替え時のデータ消失バグ修正**: 別タブに移動した際に、選択されていたパターンの入力内容がクリアされてデフォルト状態に戻ってしまう深刻なバグを修正。
- **カメラ読み込み時のMSMFエラー抑制**: Windows環境 (OpenCV) で発生していたDSHOW関連の不要な警告（`can't grab frame. Error: -1072875772`等）を強制的に抑制する処理を追加。
- **設定画面終了時のステータス復帰**: キャンセルや「×」ボタンで閉じた際も、メイン画面のステータスが正しく「待機中」に戻るよう修正。
- **撮影→編集の高速化**: プレビュースケーリング処理の最適化を行い、エディタへの画像取り込みラグを劇的に軽減。
- **UIの視認性改善**: 黒背景での文字色（システム・カメラタブ等）を明るい色に修正。チェックボックスも大型化。
- **解像度表示のクラッシュ修正**: 設定読み込み時の `AttributeError` を解消。
- **GPIO画面の NameError 修正**: 設定ダイアログ上で `tab_frame` が未定義となりクラッシュしていたバグを完全に解消。
- **重複コードの排除**: `dialogs.py` 内に存在していた `show_gpio_map` などの重複実装および日本語エンコーディングの乱れをクリーンアップ。

- **ブラックボックス化した再試行ロジックの廃止**: 射影変換失敗時に円形マスクで無理やり再試行する旧ロジック `_mask_second_try` を廃止し、検査エンジンの処理をシンプルで予測可能な形へ統一しました。

- **ストレージ容量監視の非同期化**: 容量監視処理 (`_monitor_storage`) をバックグラウンドスレッドで実行するよう改善し、監視中の画面フリーズを回避しました。また、`debug/` フォルダ内の画像もクリーンアップ対象に含めるよう拡張しました。
- **車種設定の残骸削除**: すでに廃止されていた「車種（Car Model）」に関する設定キーを `config.json` と `settings.py` の同期ロジックから完全に消去し、自動クリーンアップされるようにしました。
- **設定画面小項目のカードレイアウト化**: `inspection_app` の UI 規約に従い、設定画面（画素数・保存タブ、画像処理タブ）内の小項目グループに独立したカード枠（LabelFrame）デザインを適用し、視認性を大幅に向上させました。
- **GPIOテストのトグル化・モニタリング追加**: `inspection_app` に準拠し、出力テストボタンをトグル式（ON中オレンジ表示）に変更しました。さらに「トリガー入力」などの入力ピンのON/OFF状態を設定画面内でリアルタイムモニタリングできる機能を追加し、配線テストの利便性を向上させました。

### 修正 (Fixed)
- **GPIOテスト実行時のクラッシュ修正**: 設定ダイアログ上で出力ピンの「テスト」ボタンを押下した際、未定義のロガー（`self.logger`）を参照してアプリがクラッシュするバグを修正しました。

## [2026-04-06] v1.3.0 - 画像処理機能の統合・標準化とUI洗練

### 追加 (Added)
- **画像前処理の一括設定**: 設定ダイアログの画像処理タブに「明るさ・コントラスト・彩度・ガンマ・ぼかし・シャープ・CLAHE」の7項目を追加。これらはリアルタイム検査にも適用されます。
- **エンジンへの前処理実装**: `engine.py` に OpenCV ベースの共通前処理 `apply_preprocessing` を実装。
- **判定結果の詳細フィードバック**: メイン画面の「最終判定」エリアに、一致したマスター画像の名前とマッチングスコアをリアルタイム表示する機能を追加。
- **エディタの状態管理**: 画像が読み込まれていない状態では「切り抜き・変形」「保存・登録」ボタンを無効化（グレーアウト）し、誤操作を防止。

### 変更 (Changed)
- **UIレイアウトの統一**: 編集モード（エディタ）の操作パネルを右側に移動し、検査モードと視線を左（プレビュー）から右（操作）へ一貫した流れに統一。
- **エディタ操作のステップ化**: ボタン群を「①取得」「②変形」「③登録」の3ステップ形式に整理。直感的な作業フローを実現。
- **用語の日本語化徹底**: 「Adaptive」「Simple」「Ada Block」などの技術用語を「自動適応」「固定しきい値」「自動適応: 範囲」といった直感的な日本語表記へ刷新。
- **二値化モードの完全一元化**: 従来、画像処理タブとシステムタブで矛盾して設定できた二重構造を廃止し、3択ラジオボタンに統一。
- **前処理ロジックの標準化**: エディタ画面（PILベース）と検査エンジン（OpenCVベース）の画像補正処理を OpenCV に統一し、一貫性を向上。
- **ボタン配色の標準化**: 「保存・登録＝緑 (COLOR_OK)」、「リセット＝赤 (COLOR_NG)」等に統一。

### 修正 (Fixed)
- **カメラ接続の安定性向上 (Windows)**: DirectShow (DSHOW) バックエンドで例外が発生する問題に対し、Media Foundation (MSMF) を優先試行するように最適化。
- **設定画面内の TypeError**: `slider` 関数の引数名の不一致によるクラッシュを修正。
- **エディタのレイアウトバグ**: 配置変更時に発生していた変数参照エラー（UnboundLocalError）を修正。
- **InspectionEngine 呼び出しエラー**: インスタンス化の際の引数不足/過多を修正し、前処理プレビューが正常に機能するよう改善。
