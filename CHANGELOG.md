# CHANGELOG

## [2026-04-05] v1.1.4 - UI表示の洗練とカメラ全自動設定機能の追加

### 修正 (Fixed)
- **UIラベルの洗練**: 各設定画面および仮想GPIOパネルから `(BCM)` や `(トリガー)` といった不要な括弧表記を削除し、直感的な表示に変更。
- **GPIO設定画面の見切れの最終修正**: ラベル幅とパネル比率をさらに最適化し、右端の「削除」ボタンが隠れないようマージンを確保。
- **カメラ設定画面の見切れ修正**: プレビュー画面との干渉を防ぐため、左側設定パネルの最小幅を 360px に調整。

### 追加 (Added)
- **カメラ全項目自動設定ボタン**: 露出・フォーカス・ホワイトバランス・色相を一括で順番に自動走査・最適化する機能を実装し、カメラ設定タブの最上部に追加。

## [2026-04-05] v1.1.3 - UIレイアウトの更なる改善とロジックの適正化

### 修正 (Fixed)
- **GPIO/カメラ設定画面の見切れ修正**: 左右のパネル比率および入力欄の幅を調整し、画面が小さい環境でも「削除」ボタンや「テスト」ボタンが正しく表示されるよう改善。
- **設定後のステータス表示バグ**: 詳細設定画面を閉じた後、メイン画面のステータスバーが「設定変更中」のまま止まってしまう問題を修正。「待機中」に正しく戻るよう改善。
- **出力ピン重複チェックの仕様変更**: 仕様マッピングにおいて、「ピンが1つでも重なったらエラー」という過度な制限を緩和。「全く同じピンの組み合わせ」を持つIDがある場合のみエラーとするよう変更し、部分的なピン共有（例：ID1はピン4、ID2はピン4と5）を許可。

### 変更 (Changed)
- **仕様IDのソート表示**: 設定画面の仕様マッピングリストをIDの数値昇順（1, 2, 3...）で表示するように変更。

## [2026-04-05] v1.1.2-hotfix - Windows固有パスバグと安定性の改善 (フェーズ2/3)

### 修正 (Fixed)
- **設定キャンセル時のグローバルフリーズの修正**: 詳細設定画面を「キャンセル」またはOSの「Xボタン」で閉じた際に発生する、メイン画面のカメラ永続フリーズおよびGPIO通信の完全途絶バグを修正。画面の閉鎖手段に関わらず必ず再初期化コールバックが実行されるよう `destroy()` メソッド内にフックを統合。
- **Windows日本語パスクラッシュの修正**: Windowsの仕様により、画像やファルダのパス名に日本語が含まれた際に画像の保存や読み込みがエラーを出さずに失敗するクリティカルなバグ（OpenCV `imread` / `imwrite`）を修正。`np.fromfile` および `imdecode`/`imencode` を用いたロバストなヘルパー関数（`cv_imread` / `cv_imwrite`）へ全置換。
- **データ拡張のフリーズ防止**: 画像のデータ拡張（オーギュメンテーション）バッチ処理中にファイル権限エラーやメモリ不足が起きた際、バックグラウンドスレッドが異常終了したままになる（「実行中...」で永遠に止まる）バグを修正。安全にエラーをキャッチしポップアップ表示で復帰するよう改善。
- 詳細設定タブを開いた際のバックグラウンドGPIOリソース競合を修正（デバイスのクリーンな解放と再確保）
- 自動パラメータ調整中にTkinterのUIクラッシュを引き起こす可能性のあるスレッド安全性のバグを修正
- 調整タブの重複生成されていた Checkbutton ウィジェットを削除
- `gpio_pins` オブジェクトが存在しない場合に KeyError が発生するバグを修正
- ROIが 640x480 でハードコード保存されていた問題を修正し、解像度非依存の相対比率 (0.0~1.0) での保存に対応
- カメラ検出中の数秒のUIブロッキングを非同期スレッド化により解消
- カメラ自動調整ボタン（オートチューニング）の重複実行バッファリング問題を修正

### 追加 (Added)
- **ログの自動クリーンアップ**: 画像ファイルと同様に、「30日以上経過した古い .log ファイル」を容量監視モジュール（`_monitor_storage`）が自動的に削除する機能を追加し、完全なメンテナンスフリーを実現。
- **多重実行防止ロック**: データ拡張実行中に「データ拡張を実行」ボタンを無効化（DISABLE）し、悪意や事故による連打でマシンがクラッシュするのを防止。
- カメラの自動再接続ロジックをメインのプレビューループに追加
- バックアップ機能: 設定保存時に `config.json` を上書きする前に `config.json.bak` を自動で退避生成
- `VALID_BCM_PINS` リストに基づく、強力なGPIOピン番号の存在チェックバリデーションを導入

### 変更 (Changed)
- デバッグ画像保存によるストレージの急激な使用増加を防止するため、標準ではデバッグ画像を保存しない仕様に変更し、システムタブに「デバッグ画像を保存する」設定を新設
- 無駄なループCPU使用を防止するため、調整タブがアクティブでないときの不要なUI更新ルーチンを徹底排除

## [2026-04-04] v1.1.0 - UI/UX全面刷新および標準化

### 追加 (Added)
- **メインUIの改善**: 設定画面を開いている間、メイン画面のカメラプレビューをアイドリングさせ、処理負荷を軽減する機能を追加。
- **仕様マッピングの動的UI追加**: 設定ダイアログの「GPIOピンタブ」にて、`pin_out` 1～10の固定設定を廃止。代わりに仕様IDを動的に追加・削除できる最新の `inspection_app` 準拠のリストUIへ刷新。
- **カメラ自動調整機能の拡充**: 「画像処理タブ」（旧「調整タブ」）および「カメラタブ」に、「フォーカス」「ホワイトバランス」「色相」の自動最適化（Auto-Tune）探索ボタンを追加。
- **マウスドラッグによる直感的なマスク（ROI）指定**: 従来の6つのスライダー（左上・下など）によるポリゴンマスクを撤廃。プレビューキャンバス上でマウスをドラッグし、矩形を指定するだけで検査エリア（ROI）を設定できるよう簡略化しました。
- **仮想GPIOパネルの最新化**: Windows実行時に表示される仮想GPIOパネルのデザインを、コンパクトなLED風 UI（`inspection_app` の `widgets.py` 準拠）に近代化。
- **カメラ自動調整機能**: フォーカス、ホワイトバランス、色相、抽出露出をワンクリックで最適化する自動探索ボタンを実装。
- **カメラインデックス自動検索**: 接続されているカメラをスキャンし、有効な最初のインデックスを自動セットする機能を追加。
- **GPIOテスト機能**: 各出力ピン（OK/NGおよび仕様マッピング内のピン）の横に「テスト」ボタンを追加。クリック時に信号を0.5秒間テスト発火させる機能を実装。
- **40Pin Map 入力の連動強化**: ピン配図をクリックした際、最後に入力・クリック（FocusIn）されたBCM入力欄に自動的にピン番号がセットされるよう改善。
- **UI/UX 標準化の徹底**: メイン画面・設定画面の全てのボタン、入力欄、リストにツールチップを追加し、`common_ui_requirements.md` に完全準拠する最終調整を実施。
- **GPIO保存バリデーション**: 重複ピンや範囲外（0-40以外）の入力を保存時に検出し、警告する仕組みを導入。
- **マウスホイールスクロール**: 全ての設定タブ（カメラ、画像処理、GPIO、画素数、システム）にホイール操作による上下スクロールを同時実装。
- **包括的なツールチップ**: 設定項目の各入力フィールド（スピンボックス、スライダー等）に詳細な機能説明を表示するツールチップを全タブに配備。

### 変更 (Changed)
- **UIレイアウトの標準化**: `inspection_app` に準拠したダークテーマと、一貫性のあるカードレイアウト・スクロールパネルへ刷新。
- **GPIOラベルの適正化**: 誤解を招く表記を「トリガー入力」「OK出力」「NG出力」へ変更。
- **解像度設定タブの集約**: 撮影、プレビュー、保存、自動削除の設定を「画素数・保存」タブに集約し、スクロール可能な単一パネルへ統合。
- **プレビュー制御**: 設定画面を開いている間、メイン画面のカメラプレビューを自動停止（アイドリング）させ、カメラリソースの競合を回避。
- **デフォルト設定**: フレームレート（FPS）のデフォルト値を 10 から 5 に変更。

### 修正 (Fixed)
- 設定画面で一部のUI属性（btn_save等）が参照できず、特定の条件下で保存が失敗していた不具合を修正。
- 複数の設定変更が重なった際の UI 更新ロジックの冗長性を解消。

## [2026-04-04] v1.0.0 - 統合アプリ TM_App 初版リリース

### 追加 (Added)
- **`TM_App.py`**: 4つの独立スクリプトを1つに統合したエントリーポイントを新規作成。
- **`modules/` パッケージ**: モジュール分割による保守性向上。
  - `constants.py`: `common_ui_requirements.md` 準拠のカラーパレット・フォント定数を定義。
  - `hardware.py`: GPIO制御クラスと Windows用モックを実装（inspection_app 準拠）。
  - `settings.py`: `ConfigManager` クラスによる `config.json` 一元管理（深層マージ対応）。
  - `engine.py`: `InspectionEngine` クラス。TM3.py のコアロジック（マスク・CLAHE・二値化・輪郭抽出・射影変換・テンプレートマッチング）を移植。
  - `editor.py`: `EditorView` クラス。`image_editor_gui_ver3.py`（マスター画像エディタ）と `maseter_image.py`（データ拡張）をダークテーマのフレームに統合。
  - `dialogs.py`: `SettingsDialog` クラス。5タブ構成（カメラ/GPIOピン/調整/画素数・保存/システム）。**調整タブにリアルタイムカメラプレビューを搭載**。
  - `widgets.py`: `create_card`, `Tooltip`, `HelpWindow` など共通UIウィジェット。
  - `app.py`: `TMApp` メインクラス。ヘッダー・2モード（検査/編集）切替・操作パネル・バックグラウンドスレッドを実装。
- **`config.json`**: 旧3設定ファイル（`config_app.json`, `config_default.json`, `spec.json`）を1ファイルに統合。

### 変更 (Changed)
- UI全体を `common_ui_requirements.md` のダークグレーテーマ（`#2b2b2b` 系）に統一。
- ヘッダーに **状態別カラー**（待機:ACCENT / 検査中:ACCENT / OK:緑 / NG:赤）を適用。
- 設定ダイアログをタブ化し、変更時に「保存して閉じる」ボタンが緑色にハイライトされる **変更検知** 機能を実装。
- リアルタイム調整を「設定ダイアログ内の調整タブ」に集約（独立ウィンドウを廃止）。
- `master_{車種}/` の切替は `config.json` の `current_spec.car_model` で管理。
- データ拡張処理のスレッド化と **プログレスバー表示** を実装。
- Windows環境での **仮想GPIOパネル**（デバッグ用トリガー送信・出力確認）を統合アプリに維持。

### 新機能・UI拡充 (Added - UI & Editor)
- **カメラ詳細設定機能**: ダイアログの「カメラタブ」に フォーカス・露出・ゲイン・明るさ・コントラスト 等のプロパティ調整スライダーを追加し、プレビューへの**リアルタイム連動**を実装。
- **GPIO 40Pin Map インタラクティブ設定**: `inspection_app` の標準UIに準拠し、タブ内の「40ピン配線図」をクリックすることで自動的にピン番号を入力欄へセットする機能を実装。
- **マスター画像管理機能 (CRUD)**: `editor.py` 内にマスター画像管理を統合。
  - **新規登録**: 現在編集中の画像を「クラス名」付きで `master_image/source/{クラス名}` に直接保存可能に。
  - **削除機能**: 登録済みのマスター画像（source）を選択して削除できるダイアログを実装。
  - **データ拡張の完全統合**: `master_image.py` で行っていた処理（回転・拡縮・透視変換・ノイズ等）をエディタ内に取り込み、UIから直接増幅を実行できるパイプラインを確立。

### 廃止 (Deprecated)
- `TM3.py`, `realtime_camera_ver7.py`, `image_editor_gui_ver3.py`, `maseter_image.py` は旧バージョンとして別管理に移動。
- `config_app.json`, `config_default.json`, `spec.json` は `config.json` に統合されたため不要。
