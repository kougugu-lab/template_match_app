# AI テンプレートマッチング検査システム

**バージョン**: 1.3.1  
**最終更新**: 2026-04-07

## 概要

カメラ映像から指示書（ワーク）を検出し、テンプレート画像と照合して車種・仕様を判定するAI検査アプリケーションです。

---

## 動作フロー

1. **起動**: `config.json` からすべての設定を読み込み
2. **待機**: カメラプレビューを表示しながら GPIO `pin_Start` トリガーを待機
3. **検査開始**: トリガー入力 → フレームキャプチャ
4. **前処理**: マスク → 各種画像補正（明るさ/コントラスト等） → 二値化 (固定/自動適応/動的)
5. **ワーク検出**: 輪郭抽出 → 射影変換（歪み補正）
6. **テンプレート照合**: マスター画像フォルダとパターンマッチング
7. **判定・出力**: OK/NG判定。一致箇所（マスター名）とスコアを表示 → GPIO出力 → 画像保存 → ログ記録
8. **次サイクルへ**: 待機状態に戻る

## アプリケーションの機能

### 検査モード
- **検査プレビュー**: カメラ映像、検査状況（OK/NG）を表示
- **最終判定の詳細表示**: どのマスター画像に一致したか、またそのマッチングスコアをリアルタイムで印字
- GPIOトリガー待機 → 撮影 → テンプレートマッチング → OK/NG判定
- NG判定時は `pin_NGlog` 出力がONになります（「ブザー停止」で解除）
- NG履歴はパネルに一覧表示。蓄積されたら「履歴リセット」で消去できます
- **右パネルに統一**: 操作・ステータス情報は常に右側に集約されています。

### 編集モード
- **3ステップ・ウィザード形式**: 「①画像の取得」「②切り抜き・変形」「③保存・登録」の順に従うだけでマスターを作成可能。
- **誤操作防止機能**: 画像が読み込まれていない状態では、変形や登録のボタンが自動的に無効化（グレーアウト）されます。
- **マスター画像エディタ**: カメラ画像や既存画像を読み込み、射影変換・トリミング・画像処理を通じた加工と保存。
- **マスター画像管理**:
  - 現在の編集画像を新しいマスター画像として、仕様ごとのフォルダに直接登録。
  - 不要になった既存のマスター画像を一覧から削除可能。
- **データ拡張 (Augmentation) の統合**: 回転・拡縮・透視変換・明るさ調整・ノイズ等の一括生成をエディタ内で実行可能。
- **UIレイアウトの共通化**: メイン画面と同様に、操作メニューは右側に配置されます。

### 設定ダイアログ（詳細設定ボタンから起動）

| タブ | 内容 |
|--|--|
| カメラ | カメラIndex、解像度設定、**高度なカメラプロパティ調整（Focus/Exposure/Gain等）**、リアルタイムプレビュー |
| GPIOピン | 全ピン番号、仕様マッピング表、**40Pin Map を使用したインタラクティブなピン割り当て機能** |
| 調整 | しきい値・マスク・射影変換サイズをリアルタイムのカメラプレビューを見ながら調整 |
| 画素数/保存 | 各解像度・容量自動管理、結果保存先ディレクトリ設定 |
| システム | 処理フラグのON/OFF、車種リスト管理 |

## config.json 設定項目

### `camera` セクション

| キー | 説明 | デフォルト |
|--|--|--|
| `index` | カメラインデックス番号 | `0` |
| `resolution` | 撮影解像度 | `"1920x1080"` |
| `preview_res` | プレビュー解像度（`"プレビューなし"` で無効） | `"640x480"` |
| `focus`, `exposure` | カメラハードウェアプロパティ（一部代表例） | 機種依存 |

### `gpio_pins` セクション

| キー | 説明 |
|--|--|
| `pin_Start` | 検査開始トリガー入力ピン |
| `pin_out1`〜`pin_out10` | 仕様出力ピン |
| `pin_OKlog` | OK確認SiO出力ピン |
| `pin_NGlog` | NG確認SiO出力ピン |

### `image_processing` セクション

| キー | 説明 | デフォルト |
|--|--|--|
| `threshold` | 固定しきい値 | `30` |
| `threshold_mode` | `"simple"` (固定) / `"adaptive"` (自動適応) / `"dynamic"` (動的割合) | `"simple"` |
| `ada_block` | 自動適応時の処理範囲（奇数） | `11` |
| `ada_c` | 自動適応時の定数C | `2` |
| `white_ratio` | 動的割合モードの目標白面積率 (%) | `3` |
| `brightness` 〜 `sharpen` | 明るさ・コントラスト等の補正係数 | `1.0` / `0.0` |
| `affine_h_mm` / `affine_w_mm` | 射影変換後の実寸サイズ (mm) | `50` / `40` |
| `filter_min_len` 〜 `filter_max_area` | 輪郭フィルタの周長・面積範囲 | — |
| `decision_threshold` | マッチングスコアの合否判定しきい値 | `0.8` |

### `flags` セクション（処理フラグ）

| フラグ | 説明 |
|--|--|
| `CLAHE_FLAG` | 輝度正規化 (CLAHE) |
| `THRESHOLD_FLAG` | 動的しきい値計算 |
| `ADAPTIVE_FLAG` | 適応二値化モード |
| `CONTOURS_FLAG` | 輪郭抽出・射影変換 |
| `MASK_SECOND_FLAG` | マスク後の2回目再試行 |
| `SIO_FLAG` | SiO信号待ち |
| `LENGTH_FILTER_FLAG` | 周長フィルタ |
| `AREA_FILTER_FLAG` | 面積フィルタ |

## 新しい車種を追加するには

1. `config.json` の `car_models` リストに車種名を追加
2. `master_{車種名}/` フォルダを作成し、仕様ごとのサブフォルダ（`"1"`, `"2"` など）を作ってテンプレート画像を保存
3. データ拡張（編集モード）でバリエーションを生成
4. 設定ダイアログの「仕様マッピング」タブで各仕様IDのGPIOピンを設定
