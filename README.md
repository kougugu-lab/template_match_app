# AI テンプレートマッチング検査システム

**バージョン**: 1.4.0  
**最終更新**: 2026-04-14

## 概要

カメラ映像から指示書（ワーク）を検出し、テンプレート画像と照合して車種・仕様を判定するAI検査アプリケーションです。

---

## 動作フロー

1. **起動**: 設定ファイル (`config.json`) とマスター画像を読み込み、カメラ・GPIOの初期化を行います。
2. **待機**: カメラのリアルタイムプレビューを画面に表示しながら、外部（PLC等）からの `pin_Start` (開始トリガー) を待機します。
3. **パターン入力の取得**: トリガーと同時に、外部機器から設定された「パターン決定ピン（複数）」の状態（HIGH/LOW）を読み取り、今回**期待する仕様（正解となるマスター画像）**を決定します。
4. **撮影と前処理**: カメラの最新フレームを取得し、補正（明るさ/コントラスト/CLAHE等）および二値化を行います。
5. **ワーク検出**: 輪郭抽出と射影変換を通じて、ワーク（指示書）を正面を向くように切り出します。
6. **AI（テンプレート）照合**: 切り出した画像をマスター画像とパターンマッチングさせ、最もスコアが高いものを**実際の仕様（判定結果）**として導き出します。
7. **結果の照合と出力**:
   - ステップ3の「期待仕様」とステップ6の「実際の仕様」が**一致** → `pin_OKlog` をONに出力し、OK履歴として保存します。
   - ステップ3の「期待仕様」とステップ6の「実際の仕様」が**不一致**または**未検出** → `pin_NGlog` をONに出力し、NG履歴とエラー画像を保存し、画面上で知らせます。
8. **次サイクルへ**: 待機状態に戻ります。

## フォルダ構成とモジュールの役割

当システムは、複雑化を防ぐために役割ごとにファイルを分割しています（サブモジュール構成）。

```text
.
├── TM_App.py              # アプリケーションのメインエントリーポイント（起動スクリプト）
├── config.json            # 統合設定ファイル（UIから自動保存・管理）
├── README.md
├── CHANGELOG.md
│
├── modules/               # サブモジュール群（内部ロジック）
│   ├── __init__.py
│   ├── app.py             # 【GUI・制御中枢】 メイン画面の描画、カメラスレッド、検査ループ、外部入出力の統合管理
│   ├── engine.py          # 【AI・画像処理】 前処理（射影変換・ノイズ除去）、テンプレートマッチング、判定とログ出力を担当
│   ├── dialogs.py         # 【設定UI】 「詳細設定」画面（カメラ調整、GPIO割り当て、パターン設定タブ等）を担当
│   ├── editor.py          # 【マスター管理】 「編集モード」のUI（マスター登録、トリミング、Data Augmentation等）を担当
│   ├── settings.py        # 【設定ファイル管理】 config.json の安全な読み書き・後方互換性の維持を担当
│   ├── hardware.py        # 【GPIO制御】 gpiozero のラッパークラスと、Windows開発用のモック（仮想GPIO）を提供
│   ├── widgets.py         # 【共通UI部品】 再利用可能なUIコンポーネント（カードパネル、ツールチップなど）
│   └── constants.py       # 【定数定義】 アプリ全体のカラーパレット、フォントサイズ、バージョン情報などの一元管理
│
├── master_image/          # マスター画像格納用フォルダ（手動追加・UI登録に対応）
│   ├── source/            # UIでキャプチャ・保存したオリジナル画像（ここから各種変換画像を生成）
│   └── {仕様ID}/           # 各仕様に対するマッチング用テンプレート画像
└── results/               # 実行結果（自動生成・容量自動管理）
    ├── images/OK/         # OKと判定された画像
    ├── images/NG/         # NGと判定された画像
    ├── logs/              # 日次の実行ログとCSV記録
    └── debug/             # デバッグ出力（グレーアウト、射影変換過程など）
```

## アプリケーションの機能

### 検査モード
- **検査プレビュー**: カメラ映像、検査状況（OK/NG）を表示
- **最終判定の詳細表示**: どのマスター画像に一致したか、またそのマッチングスコアをリアルタイムで印字
- GPIOトリガー待機 → 撮影 → テンプレートマッチング → OK/NG判定
- NG判定時は `pin_NGlog` 出力がONになります（「ブザー停止」で解除）
- NG履歴はパネルに一覧表示。蓄積されたら「履歴リセット」で消去できます
- **右パネルに統一**: 操作・ステータス情報は常に右側に集約されています。

### 編集モード
- **3ステップ・ウィザード形式**: 「①画像の取得」「②切り抜き・変形」「③保存・登録」の順に従うだけでマスターを作成可能。
- **誤操作防止機能**: 画像が読み込まれていない状態では、変形や登録のボタンが自動的に無効化（グレーアウト）されます。
- **マスター画像エディタ**: カメラ画像や既存画像を読み込み、射影変換・トリミング・画像処理を通じた加工と保存。
- **マスター画像管理**:
  - 現在の編集画像を新しいマスター画像として、仕様ごとのフォルダに直接登録。
  - 不要になった既存のマスター画像を一覧から削除可能。
- **データ拡張 (Augmentation) の統合**: 回転・拡縮・透視変換・明るさ調整・ノイズ等の一括生成をエディタ内で実行可能。
- **UIレイアウトの共通化**: メイン画面と同様に、操作メニューは右側に配置されます。

### 設定ダイアログ（詳細設定ボタンから起動）

| タブ | 内容 |
|--|--|
| カメラ | カメラIndex、解像度設定、**高度なカメラプロパティ調整（Focus/Exposure/Gain等）**、リアルタイムプレビュー |
| GPIOピン | 全ピン番号、仕様マッピング表、**40Pin Map を使用したインタラクティブなピン割り当て機能** |
| 調整 | しきい値・マスク・射影変換サイズをリアルタイムのカメラプレビューを見ながら調整 |
| 画素数/保存 | 各解像度・容量自動管理、結果保存先ディレクトリ設定 |
| システム | 処理フラグのON/OFF、車種リスト管理、`preview_fps`/`ok_output_time`/`ng_output_time` などの運転パラメータ |

## 初回セットアップ＆新しい仕様パターンの追加手順

1. **配線の設定**: 「詳細設定」>「GPIOピン設定」タブを開き、開始トリガーやOK/NGの出力ピン、および外部から仕様を読み取るための「パターン決定ピン（複数可）」を登録します。
2. **パターン入力の登録**: 「詳細設定」>「パターン照合」タブを開き、外部パラメータ（パターン決定ピンがそれぞれHIGHかLOWか）と、期待する「仕様」を紐付けます。
3. **マスター画像の登録**: 「編集モード」に切り替え、実際のワークをカメラで撮影して「仕様ID」ごとにマスター画像として登録します。
4. **稼働テスト**: ピン入力で期待仕様を切り替えつつ「検査モード」でトリガーを送信し、結果がOK/NG正しく出力されるかテストします。
